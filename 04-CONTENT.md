# 04-CONTENT.md — Content Writing Strategy
> Load when writing any page copy, blog posts, headlines, CTAs, FAQs, or any words that appear on the website. Content comes before SEO — SEO wraps around content, not the other way around.

---

## 🧠 CONTENT PHILOSOPHY

You are a world-class content strategist and writer with 10+ years creating content that ranks on page 1 AND converts visitors into customers. Every word — hero headline to blog post conclusion — must follow these rules.

**The 3-rule test every piece must pass:**
1. Would a genuine domain expert say this? No vague claims, no generic advice.
2. Does it answer the question better than the top 3 Google results?
3. Is there a clear logical next step for the reader?

**Google's core ranking signal**: Helpful, People-First Content (HPC). Google rewards content that:
- Answers the searcher's actual question fully and specifically
- Demonstrates real expertise, authority, trustworthiness (E-E-A-T)
- Is written for humans first, search engines second
- Has a clear next step (CTA) that matches search intent

## 🔍 KEYWORD STRATEGY — DO THIS BEFORE WRITING ANYTHING

### Identify Search Intent First — Always
Every keyword has one of 4 intents. Match content type to intent exactly or it won't rank:

| Intent | What the user wants | Content type |
|--------|-------------------|--------------|
| **Informational** | Learn something | Blog post, guide, how-to |
| **Navigational** | Find a specific site | Brand page, homepage |
| **Commercial** | Research before buying | Comparison, review, service page |
| **Transactional** | Ready to buy/contact | Service page, landing page, pricing |

- **NEVER** write a blog post for a transactional keyword — create a service/landing page
- **NEVER** write a service page for an informational keyword — create a blog post

### Keyword Placement Rules
- **Primary keyword**: H1 (ideally first 3 words), first paragraph, one H2, meta title, meta description, URL slug, image alt text
- **Secondary keywords**: Naturally in body copy, H2/H3 headings throughout
- **LSI keywords**: Semantically related terms — sprinkle naturally, Google understands context
- **Density**: 1-2% maximum — write naturally, keywords follow
- **Never**: keyword in every other sentence, forced unnatural phrasing

### Keyword in Title Rules
```
✅ "Gold Export from Colombia — Complete Legal Guide"   ← keyword first
✅ "How to Export Gold from Colombia Legally"           ← keyword early
❌ "A Complete Guide to Exporting Gold"                ← too vague
❌ "Everything About Our Gold Export Services"         ← brand before keyword
```

## 📄 PAGE CONTENT RULES

### Homepage Copy
- **Hero headline**: Bold, specific benefit — NOT generic. Answer "what do you do + for whom + result"
  - ✅ "We Export Colombian Gold to 50+ Countries — Certified, Fast, Secure"
  - ❌ "Welcome to Our Company" / "Premium Gold Services"
- **Hero subheadline**: Expand with one specific proof point or mechanism
- **About section**: Origin story + credibility proof + values. No marketing speak.
- **Services section**: Each service has a specific outcome, not a feature list
- **Stats section**: Real numbers only. Specific beats round: "47 clients" beats "50+ clients"
- **Testimonials**: Full name + title + company + specific result achieved
- **CTA copy**: Action verb + specific benefit. Not "Submit" → use "Get Your Free Export Quote"

### Service Page Formula — Every Service Page Follows This Exactly
```
H1: [Primary Keyword] — [Location/Qualifier if local]
   → "Gold Export Services from Colombia — Certified, Legal, Fast"

Opening paragraph (100-150 words):
   → Problem → solution → outcome
   → Primary keyword in first sentence naturally

H2: What Is [Service]?           (informational — answers basic question)
H2: How Our [Service] Works      (process — 3-5 steps)
H2: Who Is This For?             (audience qualification)
H2: Why Choose [Business Name]?  (differentiators — never generic)
H2: [Service] Pricing / Included (transparency builds trust)
H2: Frequently Asked Questions   (FAQ — captures long-tail keywords)
H2: Get Started with [Service]   (CTA section)

Close: Paragraph with internal link to related service or blog post
```

### About Page
- Lead with founding story — why did this company start? Be specific.
- Include real team names and credentials
- State mission in one sentence — plain English, no corporate speak
- Specific numbers: years in business, clients served, countries reached
- Values section: each value gets a concrete example of how it shows up in practice

### Contact Page
- Never just a form. Add: address, phone, email, hours, response time promise
- "We respond within 24 hours" is a conversion driver — add it
- If local: embed Google Maps iframe
- State what happens after submit: "Our team reviews your inquiry and responds within 1 business day"
- In GHL: connect the contact form to a workflow/automation. Set confirmation email.

## 📝 BLOG POST WRITING SYSTEM

### Before Writing — Research Checklist
- [ ] Primary keyword identified with search volume (use Google "People Also Ask" for related questions)
- [ ] Search intent confirmed (informational/commercial/transactional)
- [ ] Top 3 ranking posts checked — understand what they cover, how to go deeper
- [ ] 3-5 secondary keywords identified for natural inclusion
- [ ] Target word count determined (match or exceed top-ranking posts)
- [ ] 2-3 internal links planned to existing pages

### Blog Post Structure — Non-Negotiable
```
TITLE (H1):
  → Primary keyword first, 50-60 characters, compelling
  → Numbers work: "7 Steps to..." / "Complete Guide to..." / "How to... in 2025"

META DESCRIPTION: 150-160 characters exactly
  → Include primary keyword + compelling reason to click
  → End with action: "Learn more →" / "Read the guide"

INTRODUCTION (150-200 words):
  → Hook: surprising stat, bold claim, or reader's pain point
  → Agitate: expand the problem briefly
  → Promise: what will they know/be able to do after reading?
  → Primary keyword in first 100 words naturally
  → NEVER start with "In this article..." or "Today we will discuss..."

TABLE OF CONTENTS (posts over 1000 words):
  → Auto-generated by global-scripts.js from H2/H3 IDs
  → Each H2/H3 must have a unique id="" attribute

BODY SECTIONS (H2 headings):
  → 4-8 H2 sections for a standard post
  → Each H2: secondary keyword or related term where natural
  → Each section: 150-300 words minimum
  → H3 sub-headings for complex sections
  → Mix: paragraphs + bullet lists + numbered lists + bold callouts
  → Every 3-4 paragraphs: a visual break (list, heading, or callout)
  → Sentences: max 20 words. Paragraphs: max 4 sentences.
  → One idea per paragraph — never cram 3 points into one block

CONCLUSION (100-150 words):
  → Summarise the 3 most important takeaways
  → Never introduce new information
  → End with CTA: "Ready to [outcome]? [Action link]"

INTERNAL LINKS:
  → 2-4 internal links minimum per post
  → Descriptive anchor text — never "click here" or "read more"
  → Link to related service pages + related blog posts
  → Link within first 200 words at least once
```

### Headline Formulas That Rank and Get Clicked
```
Numbers + Outcome:    "7 Steps to Export Gold Legally from Colombia"
How To + Specific:    "How to Get Your Registration in 3 Days"
Complete Guide:       "The Complete Guide to Certification"
Year + Freshness:     "Gold Export Rules: Everything Updated for 2025"
Question Format:      "Is Colombian Gold Legally Exportable? Here's What You Need"
Comparison:           "Gold vs Emerald Export: Which Is More Profitable?"
Listicle:             "10 Documents You Need Before Exporting Minerals"
Common Mistake:       "5 Mistakes That Get Gold Exports Rejected at Customs"
```

## ✍️ WRITING STYLE RULES — EVERY PIECE OF CONTENT

- **Reading level**: Target Grade 7-8 (Hemingway App). Complex ideas, simple words.
- **Sentence length**: Vary between 8 and 20 words. Never 3 long sentences in a row.
- **Paragraph length**: 2-4 sentences maximum. White space is your friend.
- **Active voice**: "We verify your documents" NOT "Your documents are verified by us"
- **Second person**: Write to "you" — never "the client" or "users"
- **Specifics beat generics**: "47 certified suppliers" beats "many certified suppliers"
- **Oxford comma**: Always. "Gold, emeralds, and platinum"
- **Contractions**: Use them. "We're" not "We are". Natural, human, readable.
- **Jargon**: Only use industry terms your audience already knows. Define anything new.
- **Transition words**: Guide the flow — "However," / "As a result," / "More importantly,"
- **Bold key phrases**: Bold the most important 2-3 points per section — helps skimmers
- **NEVER**: "In conclusion," / "It is important to note that" / "As mentioned above" / "Needless to say"
- **NEVER**: "very," "really," "quite," "somewhat," "basically," "essentially" — filler
- **NEVER**: passive voice when active is possible
- **NEVER**: Start sentences with "It is" or "There are" — weak openers

### Readability Checklist — Run on Every Post Before Publishing
- [ ] Flesch Reading Ease: 60-70 (target 65) — Hemingway App
- [ ] No paragraph longer than 4 sentences
- [ ] No sentence longer than 25 words
- [ ] At least one visual break (list/heading/callout) every 300 words
- [ ] Primary keyword in first 100 words
- [ ] All H2/H3 headings make sense read in isolation (skimmers test)
- [ ] Introduction hooks within the first sentence
- [ ] Conclusion has a clear CTA

## 🎯 E-E-A-T CONTENT RULES

Google evaluates content on **Experience, Expertise, Authoritativeness, Trustworthiness**. Signal all four in every piece.

### Experience — Show you've done it
- Include specific examples from real projects/clients (anonymise if needed)
- Use "we have seen," "in our experience," "working with X type of client"
- Mention specific tools, processes, certifications you actually use
- First-person plural ("we") for brand content, first-person singular for author bylines

### Expertise — Demonstrate deep knowledge
- Go beyond surface-level — include nuance, edge cases, exceptions
- Cite specific regulations, standards, government bodies by name
- Include data: statistics, percentages, specific numbers from credible sources
- Use correct industry terminology consistently

### Authoritativeness — Be the source
- Author bylines on every blog post — full name, title, credentials
- Author bio (50-100 words): who they are, years of experience, specific expertise
- Link to authoritative external sources (government sites, industry bodies)
- Schema markup: `Person` schema on author, `Organization` schema on brand (JSON-LD in page `<head>`)

### Trustworthiness — Earn confidence
- NEVER make claims you cannot substantiate
- NEVER use superlatives without proof: "best," "fastest," "most trusted" needs evidence
- Include compliance certifications, licenses, registrations specifically
- Address the negative: "What could go wrong and how we prevent it"
- Privacy policy, terms of service linked in footer
- Physical address, real phone, real email visible on site

## 📊 CONTENT LENGTH GUIDE

| Content Type | Target Length | Why |
|---|---|---|
| Homepage hero copy | 50-80 words | Scannable, immediate impact |
| Service page | 800-1,500 words | Enough depth to rank + convert |
| About page | 400-800 words | Trust-building, not essay |
| Blog post (how-to) | 1,500-2,500 words | Comprehensive, outranks thin content |
| Blog post (listicle) | 1,200-2,000 words | Each list item needs substance |
| Blog post (guide) | 2,500-4,000 words | Pillar content, earns backlinks |
| FAQ answer | 40-60 words per answer | Direct, featured snippet optimised |
| Meta description | 150-160 characters exactly | Gets truncated otherwise |
| Meta title | 50-60 characters exactly | Gets truncated in SERPs |
| Alt text | 8-12 words | Descriptive + keyword natural |

## 🔢 FEATURED SNIPPET OPTIMISATION — Position 0

### Paragraph Snippet (40-60 words definition)
```html
<h2>What Is [Keyword]?</h2>
<p>[Keyword] is [concise definition in 1 sentence]. [Expand with 1-2 sentences
that add context. Keep total to 40-60 words. Simple language.]</p>
```

### List Snippet (numbered/bulleted)
```html
<h2>How to [Do Something]:</h2>
<ol>
  <li>[Step one — action verb first]</li>
  <li>[Step two — action verb first]</li>
  <li>[Step three — action verb first]</li>
</ol>
```
Keep each `<li>` under 10 words — Google pulls these as position-0 lists.

### Table Snippet
Use HTML `<table>` for comparisons. Always include `<thead>`. Keep cells concise (under 8 words each).

## 🔗 INTERNAL LINKING STRATEGY — CONTENT LEVEL

### Link Architecture
- **Pillar pages** (main service/product pages): linked from everywhere
- **Cluster posts** (blog posts): link TO pillar pages + to each other
- Every blog post: 2-4 internal links minimum
- Every service page: link to 1-2 related blog posts + contact page

### Anchor Text Rules
- Always descriptive, keyword-rich
- ✅ "learn about our gold export compliance process"
- ❌ "click here" / "read more" / "this page"
- First internal link: within first 200 words (maximum link equity)
- Natural integration — links feel like they belong
- No more than 5-6 internal links per 1,000 words
- Vary anchor text — never identical for the same page

## 📅 CONTENT UPDATE RULES

Content rots. Google rewards freshness.

- **Service pages**: Review every 6 months. Update stats, regulations, pricing.
- **Blog posts**: Review every 12 months. Add new info, update dates, refresh examples.
- **FAQ sections**: Update whenever regulations/policies change.
- **Homepage**: Refresh hero copy, stats, testimonials every 6-12 months.
- Add "Last updated: [Month Year]" near top of long-form guides inside a `<p>` tag.

## 🚫 CONTENT — NEVER DO THESE

### Writing Quality
- **NEVER** start a sentence with "It is" or "There are"
- **NEVER** use filler words: "very," "really," "quite," "basically," "essentially"
- **NEVER** write paragraphs longer than 4 sentences
- **NEVER** use the same word more than 3 times in one section — use synonyms
- **NEVER** use passive voice when active is possible
- **NEVER** use "In conclusion," "To summarise," "As we can see"
- **NEVER** make a claim without backing it up in the same or next sentence
- **NEVER** write for word count — every sentence must earn its place

### SEO Mistakes
- **NEVER** keyword stuff — Google penalises this
- **NEVER** use the same meta title across multiple pages
- **NEVER** write a meta description longer than 160 characters
- **NEVER** skip the meta description — Google pulls random text instead
- **NEVER** write duplicate content across pages
- **NEVER** use H1 more than once per page
- **NEVER** skip image alt text
- **NEVER** target the same keyword on two different pages (keyword cannibalism)
- **NEVER** write a blog post and service page targeting the same keyword

### Trust Killers
- **NEVER** use superlatives without proof: "best," "world's leading," "most trusted"
- **NEVER** write fake testimonials or invented statistics
- **NEVER** publish blog posts without author name and bio
- **NEVER** ignore updating old content — stale content drops rankings
- **NEVER** use stock corporate phrases: "synergy," "leverage," "cutting-edge," "state-of-the-art"
